# MOEX News Forecast System

Streamlit-приложение для анализа новостей и прогнозирования значимых движений акций Московской биржи.

Приложение показывает:

- новостные сигналы по акциям MOEX на горизонте 24 часа;
- вероятность сильного движения после новости;
- отдельную визуальную вкладку прогноза по тикеру;
- график котировок со стрелками роста и снижения;
- полную таблицу сигналов, отсортированную от новых к старым;
- понятную расшифровку метрик модели;
- сравнение моделей, которые проверялись в 24ч-постановке.

Основной экран приложения — вкладка `Новости 24ч`. В ней используется модель `HistGradientBoosting` для поиска движения от 2% в течение 24 часов после новости. Вторая вкладка `Прогноз по тикеру` показывает ту же новую модель в более наглядном формате с индикатором и графиком. Старая модель из предыдущей постановки не удалена, но перенесена в архив: `archive/old_main_model_20260517/final_model_outputs/`.

## Запуск

```bash
python3 -m venv .venv
source .venv/bin/activate
python3 -m pip install -r requirements.txt
streamlit run app.py
```

По умолчанию Streamlit откроет приложение на `http://localhost:8501`.

## Данные

Основные пути заданы в `src/config.py`.

Файлы данных не нужно хранить в git. Они добавлены в `.gitignore`, потому что занимают много места и относятся к воспроизводимым артефактам.

При первом запуске без данных приложение покажет экран первичной загрузки. В нём можно выбрать начальную дату и загрузить:

- часовые свечи MOEX в `data/all_1h_stocks_2020_2026.csv`;
- raw-новости в `data/news_raw_2020_2026_all_sources.csv`;
- признаки новостей в `data/news_features_2020_2026.parquet`.

Для загрузки новостных признаков нужны интернет и доступ к моделям Hugging Face: `DeepPavlov/rubert-base-cased-sentence` и `mxlcw/rubert-tiny2-russian-financial-sentiment`. Первый полный запуск с 2020 года может идти долго, потому что приложение скачивает новости, загружает transformer-модели и считает эмбеддинги.

Новостные источники парсятся независимо друг от друга. Если один сайт временно не отвечает или отдаёт таймаут, он будет пропущен, а сбор продолжится по остальным источникам.

В интерфейсе обновление новостей по умолчанию работает в быстром режиме без пересчёта RuBERT/sentiment-признаков, потому что финальная 24ч-модель использует рыночный контекст в момент новости. Полный пересчёт тяжелых текстовых признаков можно включить отдельным чекбоксом `Считать RuBERT/sentiment признаки`, но он работает заметно дольше.

Для полноценной работы интерфейса используются:

- `data/all_1h_stocks_2020_2026.csv` — часовые свечи MOEX;
- `data/news_raw_2020_2026_all_sources.csv` — исходные новости;
- `data/news_features_2020_2026.parquet` — новости с признаками;
- `data/moex_external_factors.csv` — внешние рыночные факторы, создаются автоматически перед обучением, если файла нет;
- `data/final_news_24h_model_outputs/news_24h_results.csv` — метрики финальной новостной 24ч-модели;
- `data/final_news_24h_model_outputs/news_24h_test_predictions.csv` — сигналы финальной новостной 24ч-модели;
- `data/final_news_24h_model_outputs/news_24h_signal_slices.csv` — качество сигналов при разных порогах;
- `data/final_news_24h_model_outputs/models/` — сохранённые event- и direction-модели для 24ч-постановки.

Файлы в `data/final_news_24h_model_outputs/` не скачиваются из внешнего источника. Их можно получить после загрузки котировок и новостных признаков командой:

```bash
python scripts/train_main_news_24h_model.py
```

Эта команда строит датасет только по новостным часам, берёт следующий час после публикации как момент сигнала и проверяет, было ли движение не менее 2% в течение следующих 24 часов. В качестве основной конфигурации используется `price_only_on_news + HistGradientBoosting`. Результаты сохраняются в `data/final_news_24h_model_outputs/`.

Старая модель из 4-часовой постановки лежит в `archive/old_main_model_20260517/final_model_outputs/`. Она нужна только как резерв и исторический baseline, основной интерфейс её больше не использует.

Если результатов или сигналов 24ч-модели ещё нет, интерфейс запустится и покажет инструкцию с командой обучения.

## Основные модули

`app.py` отвечает за Streamlit-интерфейс, вкладку новостных сигналов, вкладку сравнения моделей и обновление данных.

`src/data_loader.py` загружает котировки, новости, результаты экспериментов и сигналы модели.

`src/market_data.py` обновляет часовые свечи через MOEX ISS.

`src/news_data.py` собирает и нормализует новости из подключённых источников.

`src/news_features.py` строит текстовые, эмбеддинговые и sentiment-признаки.

`src/news_pipeline.py` обновляет raw-новости и добавляет признаки только для новых строк.

`src/predictions.py` остался от предыдущей версии интерфейса и хранится для совместимости со старым контуром.

`src/visualization.py` строит графики, карточки прогнозов и таблицы сигналов.

`scripts/run_diploma_experiments.py` запускает старый экспериментальный контур для двух задач: `event_detection` и `direction_prediction`. Сейчас он не используется основным интерфейсом, но оставлен для воспроизводимости прошлых результатов.

`scripts/train_main_news_24h_model.py` обучает финальную новостную 24ч-модель. Она отвечает на вопрос: было ли после новости по тикеру сильное движение цены в течение следующих 24 часов. В текущей версии основной порог движения равен 2%, а лучшая практическая конфигурация — `price_only_on_news + HistGradientBoosting`.

`updates/` хранит дополнительные исследовательские ноутбуки, скрипты и артефакты моделей. Вкладка `Модели 24ч` использует таблицы сравнения из `updates/model_outputs/news_24h_event_experiment_strong/`, чтобы показать, почему основной была выбрана модель `price_only_on_news + HistGradientBoosting`.

## Служебные скрипты

`scripts/build_moex_factors.py` загружает внешние рыночные факторы MOEX, сейчас индексы `IMOEX` и `RTSI`, и сохраняет их в `data/moex_external_factors.csv`. В обычном сценарии вручную запускать его не нужно: перед обучением `scripts/run_diploma_experiments.py` сам создаст этот файл, если он отсутствует. Ручной запуск нужен только если хочется заранее обновить факторы отдельно от обучения.

`scripts/build_news_features.py` полностью пересчитывает признаки новостей из raw-файла `data/news_raw_2020_2026_all_sources.csv` и сохраняет результат в `data/news_features_2020_2026.parquet`. Обычно это не нужно делать вручную, потому что интерфейс умеет обновлять новости инкрементально; скрипт полезен, если parquet с признаками удалён или нужно пересобрать его заново.

`scripts/run_diploma_experiments.py` запускает старый сценарий обучения и оценки моделей для задач `event_detection` и `direction_prediction`. Он сохранён как воспроизводимый baseline, но основной интерфейс теперь работает с 24ч-моделью.

`scripts/train_main_news_24h_model.py` обучает финальную модель для вкладки `Новости 24ч` и сохраняет её результаты в `data/final_news_24h_model_outputs/`. Старые файлы `data/final_model_outputs/` при этом не удаляются и не перезаписываются.

В боковой панели приложения есть кнопка `Переобучить 24ч-модель`; она запускает этот же скрипт из интерфейса и обновляет файлы модели.

```bash
python scripts/build_news_features.py
python scripts/run_diploma_experiments.py --help
python scripts/train_main_news_24h_model.py
```

Старая модель перенесена в `archive/old_main_model_20260517/final_model_outputs/` и может быть восстановлена вручную при необходимости.
